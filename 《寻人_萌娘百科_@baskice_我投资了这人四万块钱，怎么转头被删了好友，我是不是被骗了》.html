以我的角度看到的项目情况是这样的，只是个人看法，供参考：

项目的本身是为了保护原创内容及其创作者，是匿名的创作内容销售平台。这个愿景是不错的，也确实可以做成区块链去中心化的产品。

但是在和站长沟通的过程中，站长表示只是利用区块链来做防盗版，加密？我不清楚项目最初是要实现一个怎样的产品。但是目前来看也就是说整个产品实际上只是一个中心化的网站而已，和现有的内容销售平台并无区别，利用区块链只是想利用区块链的一些特性(?)做防盗版。站长为了顺应当时区块链热潮，所以采用 ICO 的方式来吸引投资。

因为站长很少正面回答问题，在重复拿某网站上写他“ 800 万”说事，强调只有 500 万，但还是可以看到一些问题：

1. 我访问了官网上所有“团队成员”的 github 和 blog 等链接，发现全部都是前端开发者，并询问是否全部都是前端开发者，站长回复的是：“之前后端全部是私有状态，我只给投资人看了”，并表示“前段时间投资人们决定彻底最小化资金装死，，我们把已经做了的开发全部从私有转公开了”将后端项目开源。然后我又访问了一下项目的 GitHub （ https://github.com/MoeCoinOne ），发现全都是 javascript css vue html 和一个 erc20 token 合约，但是作为外人非要了解项目代码也不合适，所以我没有继续追问。

2. 我从一些朋友处了解到，该项目在编写白皮书和制作官网的时候，站长曾经询问跟此项目没有任何关系并明确不参与此项目的人，是否可以把他的名字挂到自己的“团队成员”中，就此问题，站长表示“很多人觉得区块链就是骗钱”，所以没有参与。

3. 站长在知道自己没有技术，并且自己的“团队成员”不靠谱的情况下，就进行融资。站长就此问题作出肯定答复“确实有问题”

4. 既然项目跟区块链关系不大，并且没有找到任何区块链开发者，实际上也跟区块链没太大关系，为什么不采用受监管的正常融资渠道，而是采用 ICO 的形式呢？并且 ICO 的主体公司还是在融资完全完成的状态下才注册的.... 当然考虑到项目初心，只是由于后期原因区块链相关没有开发出来，这点并没有什么不妥。


总结就是：

站长并没有想骗钱，由于能力 /团队 /币价等多方面因素导致项目没有成功（其实我感觉这项目从最开始就很难成功。。。），但是在这种情况下融资，确实是一种不负责任的行为。
